Архитектура приложения DrivePro

1. Общая архитектурная модель
Приложение DrivePro построено по клиент-серверной архитектуре.
Система состоит из двух основных частей:
•	Клиентская часть (Frontend)
•	Серверная часть (Backend)
•	База данных (SQLite)
Клиент взаимодействует с сервером через HTTP-запросы. Сервер обрабатывает запросы, выполняет бизнес-логику и взаимодействует с базой данных.


2. Компоненты системы

Frontend:
•	HTML
•	CSS
•	JavaScript
Основные задачи frontend:
•	Отображение пользовательского интерфейса
•	Отправка данных форм на сервер
•	Отображение динамического контента
•	Обработка состояний (загрузка, данные, ошибка)
________________________________________

2.2 Backend
Backend реализован на Python с использованием фреймворка Flask.
Основные задачи backend:
•	Обработка маршрутов
•	Аутентификация пользователей
•	Управление сессиями
•	Обработка форм
•	Работа с базой данных
•	Формирование HTML-ответов и API-ответов

________________________________________
2.3 База данных
В качестве базы данных используется SQLite.
База данных отвечает за хранение:
•	Пользователей
•	Курсов
•	Заявок на обучение
________________________________________

3. Поток обработки запроса
1.	Пользователь открывает страницу в браузере.
2.	Браузер отправляет HTTP-запрос на сервер.
3.	Сервер обрабатывает запрос через соответствующий маршрут.
4.	При необходимости сервер выполняет запрос к базе данных.
5.	Сервер формирует ответ (HTML или JSON).
6.	Браузер отображает полученный результат.
________________________________________

4. Разделение доступа
Приложение разделено на:
•	Публичную часть (доступна всем пользователям)
•	Приватную часть (доступна только авторизованным пользователям)
Проверка доступа осуществляется через механизм сессий.
